## Featured: `attribution.sessionUrl` Setting

Controls whether Claude Code appends a `Claude-Session: https://claude.ai/code/session_...` link to git commits and PR descriptions. Added in v2.1.183 (June 19).

**Why it's useful:** Enterprise teams often don't want session URLs in their git history for privacy or compliance reasons. CI/CD pipelines using Claude Code in headless mode produce cleaner commits without dangling session links. Also useful for open-source contributors who prefer not to expose their session metadata.

**Usage:**
```bash
# Disable session URL in commits/PRs for this project
claude config set attribution.sessionUrl false

# Or in .claude/settings.json
{
  "attribution": {
    "sessionUrl": false
  }
}
```

Commits will still include the `Co-Authored-By: Claude` line but omit the session link.
