## Featured: `worktree.baseRef` Setting
When Claude Code spawns a subagent with `isolation: "worktree"`, it creates a git worktree for the agent to work in. Until v2.1.133, the worktree always branched from `origin/main` — meaning subagents could be hundreds of commits behind your working branch, silently operating on stale code.

The new `worktree.baseRef` setting fixes this:

```json
// .claude/settings.json
{
  "worktree.baseRef": "head"
}
```

- **`fresh`** (default): branches from `origin/main` — clean baseline, no local changes
- **`head`**: branches from your current HEAD — subagents see the same code you do

Use `head` when your subagents need to work on code from the current branch (the common case). Use `fresh` when you want subagents isolated from in-progress work.
