## Featured: `/goal` Command

The `/goal` command, introduced in Claude Code v2.1.139 (May 11), lets you define a completion condition and set Claude to work autonomously across multiple turns until the condition is met. A lightweight validator model checks after each step: "Has the goal been met?" If no, the main agent keeps going. If yes, the loop closes and results are reported.

**Why it matters:** Eliminates manual iteration for multi-step tasks. Ideal for migrations, backlog processing, and any work with a verifiable end state. Early adopters describe it as "a self-driving coding agent."

**Usage:**
```
/goal Migrate all React class components in src/ to functional components with hooks. All tests must pass.
```

Claude scaffolds, refactors, runs tests, fixes failures, and loops â€” tracking elapsed time, turns, and tokens. It's a soft loop with check-ins, not a runaway process. Works best when the goal has a clear, machine-verifiable success condition.
