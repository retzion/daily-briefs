## Featured: Session Forking (`/branch`)

Session forking lets you duplicate a Claude Code session's full context as an independent branch — essentially `git branch` for your conversation. The original session stays untouched while the fork gets a complete copy of the history.

**Why it's useful:** When exploring a tricky implementation, you often want to try two approaches without losing context. Instead of starting over or polluting your current session, fork it and run both paths in parallel.

**Quick usage:**
```bash
# From within a session — fork the current conversation
> /branch

# From the CLI — fork an existing session
claude --resume <session-id> --fork-session

# Practical workflow: you're mid-refactor and want to try two approaches
> /branch                    # creates a fork
# Continue approach A in the current session
# Switch to the fork for approach B in another terminal
```

Forks are independent — edits, tool calls, and context in one don't affect the other. Combine with `claude agents` to run both forks simultaneously.
