## Featured: `/pdf` Skill (PDF Operations)

The built-in `/pdf` skill tells Claude Code which library to reach for, the correct patterns for each operation, and the pitfalls no README covers — so you don't waste a turn debugging a broken PDF pipeline.

**What it does:** Covers the full PDF lifecycle — reading text, extracting tables, creating new PDFs, filling forms, merging/splitting, and OCR for scanned documents. It selects the right tool for each job: `pdfplumber` for text-based PDFs, `pytesseract` for scanned ones, `reportlab` for generation, `qpdf` for structural operations.

**Why it's useful:** Without the skill, Claude Code often guesses wrong on library choice or misses pre-processing steps (deskewing, contrast enhancement) that make OCR reliable on real-world scans. The skill encodes these decisions so the first attempt works.

**Quick usage:**
```
# Extract text and tables from a spec PDF
/pdf
"Extract all tables from spec.pdf into a CSV"

# Fill out a form PDF
/pdf
"Fill out form.pdf with: name=Acme Corp, date=2026-07-23, amount=5000"

# Convert scanned invoice to structured data
/pdf
"OCR this scanned invoice and extract line items into JSON"
```

The skill auto-detects whether a PDF is text-based or scanned and routes accordingly — no manual library selection needed.
