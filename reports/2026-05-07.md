## Featured: Managed Agents Dreaming

Announced at Code with Claude on May 6, **Dreaming** is a new research-preview capability for Claude Managed Agents. It lets agents review their past sessions — up to 100 at a time — to identify recurring mistakes, discover proven workflows, and share insights across future sessions.

**Why it matters:** Agents today start every session from scratch. Dreaming gives them a form of long-term learning — not by retraining the model, but by running an asynchronous review job that reads past sessions and an existing memory store, then surfaces patterns (e.g., "this API always returns paginated results" or "tests in this repo need the DB seeded first").

**How it works:**
1. Dreaming runs as a background job on the Claude Platform — not during active sessions
2. It reads your agent's memory store plus a batch of past session transcripts
3. It produces structured insights that are injected into future sessions as context
4. The agent effectively "learns" from experience without model fine-tuning

Currently in research preview as part of the Managed Agents public beta. Available via the Claude Platform API.
