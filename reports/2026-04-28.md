## Featured: /recap

The `/recap` command generates a summary of your current session context, designed for when you step away and return later.

**Why it's useful:** Long Claude Code sessions accumulate context that gets compressed over time. When you return after a break, `/recap` gives you (and Claude) a quick orientation — what was accomplished, what's in progress, and where things stand — without scrolling through history.

**Usage:**
```
/recap              # generates a recap of the current session
```

You can also configure automatic recaps in `/config` so Claude proactively summarizes context when you return to a dormant session. Unlike `/compact` (which lossy-summarizes and replaces history), `/recap` is read-only — it surfaces context without destroying it.

Added in v2.1.108. Pairs well with Session Memory for cross-session continuity.
