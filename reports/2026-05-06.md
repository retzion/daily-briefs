## Featured: Plugin Archives & Remote Loading (`--plugin-url`)

Claude Code v2.1.128-129 introduced two improvements that make plugin distribution much easier:

**What it is:** Plugins bundle MCP servers, skills, subagents, rules, and hooks into a portable package. Previously you needed a local directory. Now you can load plugins from `.zip` archives or directly from a URL.

**Why it's useful:** Teams can host plugin bundles on an internal server or CDN and share them with a single URL — no cloning repos, no manual setup. Great for onboarding, standardizing team workflows, or distributing custom skills across an org.

**Quick usage:**

```bash
# Load a plugin from a URL for this session
claude --plugin-url https://internal.example.com/plugins/our-team-plugin.zip

# Load a local .zip archive
claude --plugin-dir ./my-plugin.zip
```

Plugins are session-scoped by default — they don't persist unless added to your project config. Combine with the new `skillOverrides` setting (`off`, `user-invocable-only`, `name-only`) to control which plugin skills are available.
