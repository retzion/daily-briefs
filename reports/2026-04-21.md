## Featured: `/effort` Slash Command

The `/effort` command controls how much reasoning Claude applies to your prompts. With Opus 4.7's release this week, a new **xhigh** level slots between `high` and `max`, giving extended reasoning without jumping to the slowest, most expensive tier.

**Why it matters:** Most tasks don't need `max` effort (which burns tokens on deep exploration). `xhigh` is now the default on Opus 4.7 and hits the sweet spot for multi-file refactors, concurrency bugs, and long agentic runs.

**Usage:**
```
/effort              # opens interactive slider
/effort xhigh        # set directly
/effort auto         # reset to model default
claude --effort high  # set from CLI
```

Five levels available: `low` (fast, cheap) → `medium` → `high` → `xhigh` → `max` (deepest reasoning).

Sources: [Claude Code docs](https://code.claude.com/docs/en/model-config), [ClaudeLog FAQ](https://claudelog.com/faqs/what-is-slash-effort-command/)
