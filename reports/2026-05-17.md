## Featured: /init (CLAUDE.md Initialization)
The `/init` slash command generates a **CLAUDE.md** file tailored to your project by analyzing your codebase — package files, folder structure, configs, and existing docs. CLAUDE.md becomes part of Claude's system prompt, so every session starts with your project's context already loaded.

**Why it matters**: Without a CLAUDE.md, you re-explain your stack, conventions, and commands every session. With one, Claude knows your build commands, test runners, linting rules, and architecture patterns from the first message.

**What it generates** (typically 60–80 lines):
- Project summary and tech stack
- Build, test, and lint commands
- Key coding conventions and patterns

**Usage**:
```
claude
> /init
```
Claude scans your repo and produces a CLAUDE.md at the project root. Review it, add anything Claude couldn't infer (branch naming, deployment process, review requirements), and commit it to your repo so the whole team benefits.

**Pro tip**: You can also create `~/.claude/CLAUDE.md` for personal preferences that apply across all projects (preferred language, formatting style, etc.).
