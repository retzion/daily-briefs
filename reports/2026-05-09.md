## Featured: Auto Mode Configuration (`settings.autoMode`)
Auto mode lets Claude Code execute multi-step workflows without constant permission prompts. Instead of `--dangerously-skip-permissions`, auto mode uses a two-stage classifier to assess risk — safe operations proceed, risky ones get blocked or escalated.

**New in v2.1.136:** `settings.autoMode.hard_deny` adds unconditional block rules that cannot be overridden by allow exceptions. This complements the existing configuration slots:

```jsonc
// .claude/settings.json
{
  "autoMode": {
    "environment": "Internal monorepo. Staging deploy target: staging.example.com",
    "allow": ["$defaults", "deploy to staging via scripts/deploy.sh"],
    "soft_deny": ["$defaults", "never run database migrations outside db-cli"],
    "hard_deny": ["never execute rm -rf on production directories"]
  }
}
```

- **`environment`** — describes your trusted infrastructure in prose; fixes most false positives
- **`allow`** — explicit carve-outs where a built-in block matches but the action is safe
- **`soft_deny`** — custom block rules on top of defaults (overridable by allow)
- **`hard_deny`** — unconditional blocks, no override possible

Available on Max, Team, Enterprise, and API plans. Docs: [Auto Mode Config](https://code.claude.com/docs/en/auto-mode-config)
