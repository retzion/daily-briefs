## Featured: /security-review Skill

With Claude Security entering public beta and Cursor shipping its own Security Review this week, security-focused agentic workflows are having a moment. Claude Code has a built-in `/security-review` skill that performs a comprehensive security audit of your pending changes — no enterprise plan required.

**What it does:** Analyzes all uncommitted changes on the current branch for security vulnerabilities — injection flaws, auth issues, secrets exposure, insecure defaults, and other OWASP Top 10 risks. Produces a structured report with severity ratings, affected files/lines, and remediation guidance.

**Why it's useful:** Catches security issues before they reach code review. Unlike external SAST tools, it understands your full codebase context and can reason about business logic flaws, not just pattern-match known signatures.

**Usage:**
```
/security-review
```

That's it — one command, run from any branch with pending changes. It reads your diff, cross-references with the broader codebase, and reports findings inline. Pair it with a pre-push hook for continuous security checks:

```json
// .claude/settings.json
{
  "hooks": {
    "PrePush": [{ "command": "claude -p 'run /security-review and exit 1 if any HIGH severity issues found'" }]
  }
}
```
