## Featured: `/usage` Per-Category Breakdown

The `/usage` command (v2.1.149, May 22) now shows a detailed breakdown of your limit consumption by category — skills, subagents, plugins, and per-MCP-server cost — instead of just a single aggregate number.

**Why it's useful:** When you're approaching rate limits, knowing *where* your budget is going lets you make informed decisions. Maybe a chatty MCP server is burning tokens, or subagent-heavy workflows are consuming more than expected. This breakdown turns a black box into an actionable dashboard.

**Usage:**
```
/usage
```

The output shows your current period's consumption broken down by:
- Main conversation
- Skills invocations
- Subagent sessions
- Plugin calls
- Individual MCP server costs

Previously you could only see `/extra-usage` (now `/usage-credits`) for aggregate spend. The new per-category view complements it by showing *where* the tokens go.
