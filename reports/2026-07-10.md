## Featured: MCP `roots/list` Support
Claude Code v2.1.203 added support for the MCP protocol's `roots/list` method, exposing the session's working directories — including those added via `/add-dir` — to connected MCP servers. When directories change, Claude Code sends a `notifications/roots/list_changed` notification.

**Why it matters**: MCP servers can now dynamically discover which directories the session cares about, enabling smarter file watching, indexing, and context-aware tool behavior in multi-repo workspaces. Before this, MCP servers had no protocol-level way to know the session's scope.

**Example**: If you're working across a monorepo and `/add-dir ../shared-lib`, any connected MCP server (like a linter, search indexer, or custom tool) receives the updated roots list and can immediately start serving that directory — no restart or manual config needed.

This is part of the broader MCP spec (see the [MCP roots documentation](https://modelcontextprotocol.io/docs/concepts/roots)) and brings Claude Code closer to full protocol compliance.
