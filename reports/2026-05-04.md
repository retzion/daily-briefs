## Featured: `claude project purge`

New in v2.1.126, `claude project purge` is a cleanup command that deletes all Claude Code state for a project — transcripts, tasks, file history, and the project's config entry.

**Why it's useful:** Over time, projects accumulate stale session data that can slow down `/resume`, inflate disk usage, and clutter the project picker. This gives you a clean slate without manually hunting through `~/.claude/`.

**Usage:**
```bash
# Preview what would be deleted
claude project purge /path/to/project --dry-run

# Purge with confirmation prompt
claude project purge /path/to/project

# Purge without confirmation
claude project purge /path/to/project -y

# Interactive picker for which project to purge
claude project purge -i

# Purge ALL projects
claude project purge --all
```

Especially handy after finishing a large feature branch or when onboarding to a repo where a previous developer left behind heavy session state.
