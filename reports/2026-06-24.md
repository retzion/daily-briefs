## Featured: `sandbox.credentials` Setting

Added in v2.1.187 (June 23), `sandbox.credentials` is a new built-in setting that blocks sandboxed commands from reading credential files (like `~/.aws/credentials`, `~/.ssh/id_*`, `~/.config/gcloud/`) and secret environment variables — without requiring manual `denyRead` rules.

**Why it matters**: Previously, sandbox mode left credential files readable by default. Users had to manually configure `sandbox.filesystem.denyRead` entries for each sensitive path. The new setting provides one-line protection against credential exfiltration — particularly timely given the agentjacking attack disclosed this month.

**Usage**:
```json
// .claude/settings.json
{
  "sandbox.credentials": "block"
}
```

> **Note**: Full documentation is still being written ([tracking issue](https://github.com/anthropics/claude-code/issues/70440)). The setting was introduced as a direct response to credential-access concerns raised in security research.
