## Featured: `CLAUDE_CLIENT_PRESENCE_FILE` (Presence-Based Notification Suppression)

New in v2.1.181 (June 17), this environment variable lets you suppress mobile push notifications from Claude Code when you're actively at your machine.

**Why it's useful:** If you run Claude Code sessions in the background (via Remote Control, routines, or `claude agents`), you normally get push notifications on your phone when the agent needs attention. But if you're already at your desk, those pings are redundant noise. This env var solves that elegantly — point it at a marker file, and as long as that file exists, mobile notifications are suppressed.

**Usage:**
```bash
# In your shell profile (.bashrc, .zshrc, etc.)
export CLAUDE_CLIENT_PRESENCE_FILE="$HOME/.claude-presence"

# Create the file when you sit down (or automate via login hook)
touch ~/.claude-presence

# Remove it when you step away (or automate via screen lock hook)
rm ~/.claude-presence
```

Pair it with a macOS `sleepwatcher` or Linux `xss-lock` script to auto-toggle presence based on screen lock state for a fully automated setup.
