## Featured: /code-review skill

The `/code-review` skill runs a structured review of your current diff, scanning for correctness bugs and suggesting reuse, simplification, and efficiency improvements. You control the depth with an effort level:

- **Low/medium**: fewer, high-confidence findings only
- **High/max**: broader coverage, may include uncertain findings

Two powerful flags make it actionable:

- `--comment` — posts findings as inline PR comments on GitHub, so reviewers see them in context
- `--fix` — applies the suggested fixes directly to your working tree

**Usage:**
```
/code-review              # default effort review of current diff
/code-review high         # deeper scan
/code-review --fix        # review and auto-apply fixes
/code-review --comment    # post findings as PR comments
```

This is distinct from `/review` (which reviews an entire PR) and `/simplify` (which focuses only on cleanup, not bugs). `/code-review` is the go-to for catching real bugs before you push.
