## Featured: /compact (Context Management)

The `/compact` command compresses your conversation history into a summary, replacing the full transcript with a condensed version. This dramatically reduces context size while preserving the key facts Claude needs to keep working.

**Why it matters:** Claude Code re-reads the full conversation on every message. Long sessions don't just cost more tokens — they degrade response quality as earlier instructions become less reliably recalled. Auto-compact kicks in at ~80% context utilization, but by then quality has already slipped.

**Best practice:** Run `/compact` proactively at ~60% context utilization, or after completing each discrete task within a session. You can add focus instructions:

```
/compact focus on the API refactor and test failures
```

You can also add a "Compact Instructions" section to your `CLAUDE.md` to control what's always preserved — architectural decisions, active constraints, file scope:

```markdown
# Compact Instructions
Always preserve: database schema decisions, API versioning strategy, list of modified files
```

**Rule of thumb:** Every 30-45 minutes of active work, or after every major milestone (finishing a module, resolving a bug), run `/compact`. Your future self will thank you.
