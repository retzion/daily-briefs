## Featured: `claude plugin init` (Plugin Scaffolding)

New in v2.1.157, `claude plugin init` scaffolds a complete plugin in `.claude/skills/` with one command. It creates a `.claude-plugin/plugin.json` manifest and a starter `SKILL.md`, ready to customize. Plugins placed in `.claude/skills` are now auto-loaded on session start -- no marketplace install needed.

**Quick start:**
```bash
claude plugin init my-tool
# Creates .claude/skills/my-tool/ with:
#   .claude-plugin/plugin.json  (manifest)
#   SKILL.md                    (starter skill)
# On next session, loads as my-tool@skills-dir
```

Plugins can include skills, custom agents, hooks, MCP servers, LSP servers, and background monitors. Keep `plugin.json` inside `.claude-plugin/`; all other directories (commands/, agents/, hooks/) go at the plugin root. See the [plugin docs](https://code.claude.com/docs/en/plugins) for the full manifest schema.
