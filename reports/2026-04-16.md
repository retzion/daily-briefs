## Featured: Push Notification Tool — Step Away From the Terminal
Claude Code's **Push Notification tool** (shipped April 15 in v2.1.108) lets a running agent send a mobile push when it finishes a task or needs approval. It's the missing piece of the Remote Control story: you kick off a long-running change from your laptop, close the lid, and Claude pings your phone when it's time to review the PR or approve a `rm`.

**Why it's useful:** Agents that take 10+ minutes (migrations, big refactors, test-fixing loops) used to force you to babysit the terminal or write a glue script around `Monitor` to alert yourself. The push tool is first-party, zero-config beyond the toggle, and crucially, it's *model-initiated* — Claude decides when a notification is warranted rather than firing on every tool call.

**Setup:**
```
/config                           # open settings
# Set Remote Control: enabled
# Set Push when Claude decides: on
# Pair your phone in the Claude mobile app → Settings → Remote sessions
```

**What triggers a push:** task complete, permission prompt blocked on user, long-running step milestone, or explicit model decision (e.g., "tests are red and I need a steer"). You can reply to the push from the mobile app — grant permission, send a follow-up message, or hand off to another agent.

**Pairing tip:** Pair with `/loop` for scheduled checks (e.g., "every 30m, verify the canary is healthy; push me if anything degrades") — the push tool turns the loop from a polling pattern into an on-demand interrupt. ([changelog entry](https://code.claude.com/docs/en/changelog) · [feature issue](https://github.com/anthropics/claude-code/issues/29438))
