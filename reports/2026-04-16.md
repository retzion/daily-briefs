## Featured: `/ultrareview` — Cloud Multi-Agent PR Review
`/ultrareview` shipped today with Opus 4.7. It's a cloud-hosted, parallel multi-agent review pass that reads through a diff with multiple reviewer subagents specialized on different axes (correctness, design, security, test coverage), aggregates findings, and returns a structured critique — the kind of review a careful senior engineer would produce, without burning your local context.

**Why it's useful:** `/simplify` fixes *mechanical* over-engineering; `/security-review` covers vulnerability surface; `/ultrareview` is the one that catches **design-level** mistakes — wrong abstraction, missed edge cases, subtle concurrency bugs, API contracts that will bite callers. It runs in the cloud, so it doesn't pollute your current session's context window, and it's substantially deeper than an inline review because each subagent has room to think hard about one dimension.

**Usage:**
```
# Review your current branch against its base
/ultrareview

# Review a specific GitHub PR by number
/ultrareview 1247

# Review a PR in another repo you have access to
/ultrareview owner/repo#42
```

**Billing:** Pro and Max Claude Code plans get **3 free `/ultrareview` runs** to try it out; after that it bills against your plan's cloud-execution budget. Free-tier users can trigger it from the GitHub PR comment integration with `@claude ultrareview`.

**Workflow tip:** the strongest pre-merge stack right now is `/simplify` (local, fast, cleanup) → `/security-review` (local, vuln surface) → `/ultrareview` (cloud, design-level critique) → push. Each stage catches what the previous one misses, and only the last one costs cloud budget. Pair with **Push Notification** so you can fire `/ultrareview` and walk away — Claude pings you when the review is back. ([changelog](https://code.claude.com/docs/en/changelog) · [hands-on](https://www.mejba.me/blog/claude-code-ultra-review-tested))
