## Featured: Self-hosted Sandboxes

Self-hosted sandboxes let you run Claude's code execution on your own infrastructure while Anthropic handles the agent orchestration. Announced at Code with Claude London (May 19) and now in public beta, this feature is aimed at teams that can't send source code or build artifacts to third-party environments.

**Why it matters:** Enterprise adoption of agentic coding has been gated by security requirements — repos, secrets, and build tooling staying behind the firewall. Self-hosted sandboxes remove this blocker while keeping the agent loop (context management, error recovery, tool routing) on Anthropic's optimized infra.

**How it works:**
1. Deploy a sandbox provider in your infrastructure (supported: Cloudflare microVMs, Daytona, Modal, Vercel)
2. Configure the sandbox endpoint in your Claude Platform settings
3. The agent loop on Anthropic's side sends tool execution requests to your sandbox; files and network traffic stay inside your perimeter

```json
// Example: platform config pointing to a self-hosted sandbox
{
  "sandbox": {
    "provider": "cloudflare",
    "endpoint": "https://sandboxes.internal.yourco.com",
    "resource_class": "large"
  }
}
```

Docs: [Self-hosted sandboxes](https://platform.claude.com/docs/en/managed-agents/self-hosted-sandboxes)
