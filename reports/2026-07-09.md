## Featured: `--json-schema` (Structured CLI Output)

The `--json-schema` flag forces the `claude` CLI to return output conforming to a specific JSON Schema — essential for building reliable automation pipelines where you need predictable, parseable data rather than free-form text.

**Why it's useful:** When scripting with `claude -p` in headless/pipe mode, you often need structured data: file lists, analysis results, configuration objects. Without `--json-schema`, you'd need fragile regex or text parsing. With it, Claude returns valid JSON matching your schema every time.

**Quick example:**
```bash
claude -p "List the top 3 issues in this codebase" \
  --json-schema '{
    "type": "object",
    "properties": {
      "issues": {
        "type": "array",
        "items": {
          "type": "object",
          "properties": {
            "file": {"type": "string"},
            "line": {"type": "integer"},
            "severity": {"type": "string"},
            "description": {"type": "string"}
          },
          "required": ["file", "line", "description"]
        }
      }
    },
    "required": ["issues"]
  }'
```

This returns a clean JSON object with an `issues` array — ready for downstream tooling, dashboards, or CI integrations.

**Recent fix (v2.1.205):** Previously, an invalid schema silently fell back to unstructured output. Now it errors immediately, preventing silent pipeline failures. Schemas using the `format` keyword were also incorrectly rejected — now fixed.
