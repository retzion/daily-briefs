## Featured: /verify Skill
The `/verify` skill runs your app and observes its actual behavior to confirm that a code change works — going beyond type checks and test suites to test the real feature.

**Why it matters**: Passing tests doesn't mean a feature works correctly in the browser or terminal. `/verify` launches the app, exercises the golden path and edge cases, and watches for regressions — catching UI bugs, broken flows, and visual issues that static analysis misses.

**When to use it**:
- After implementing a feature, before reporting it as done
- To confirm a bug fix actually resolves the issue end-to-end
- To validate a PR's changes work in practice, not just in theory

**Usage**:
```
> /verify
```
Claude Code will identify your project type, start the dev server or app, navigate the relevant flows, and report whether the change works as intended. It handles web apps, CLIs, TUIs, and more — looking for a project-specific run skill first, then falling back to built-in patterns.

**Pro tip**: Pair `/verify` with `/review` for a thorough check — `/review` catches code quality issues, `/verify` catches behavioral ones.
