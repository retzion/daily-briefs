## Featured: `/simplify` — Post-Implementation Cleanup via 3-Agent Parallel Review
`/simplify` is a bundled Claude Code skill (ships with every session) that runs a **post-implementation code review** on whatever you just changed. It spawns **three review subagents in parallel** — one checking **reuse** (duplicated logic, existing helpers), one checking **quality** (clarity, dead code, error handling), one checking **efficiency** (perf, allocation, needless work) — then aggregates findings, applies valid fixes directly, and silently drops false positives.

**Why it's useful:** Agents over-engineer. They add null checks for impossible cases, create one-off helper classes, duplicate utilities that already exist three folders over. `/simplify` is the cleanup pass you should be running before every commit — it's cheap, it's built-in, and it only touches files in your current `git diff`.

**Usage:**
```
# Basic — reviews everything in git diff (or git diff HEAD)
/simplify

# Focus a single dimension
/simplify focus on memory efficiency
/simplify only flag duplicated logic

# If there are no git changes, it falls back to recently-modified files
```

Workflow tip: pair `/simplify` with `/security-review` right before you stage — the two cover complementary ground (code quality vs. vulnerability surface) and together replace the older monolithic `/review`. ([ClaudeFast guide](https://claudefa.st/blog/guide/mechanics/simplify-batch-commands) · [code-simplifier plugin](https://github.com/anthropics/claude-code/blob/main/plugins/pr-review-toolkit/agents/code-simplifier.md))
