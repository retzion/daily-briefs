## Featured: PR Activity Subscription (`subscribe_pr_activity`)

Claude Code can watch a pull request and automatically respond to CI failures and review comments — no polling or sleep loops required. When you ask Claude to "watch" or "babysit" a PR, it calls `subscribe_pr_activity` and then sits idle until GitHub webhook events arrive as `<github-webhook-activity>` messages, waking the session to investigate and fix issues.

**What it does:**
- Monitors CI/CD failures: reads error output, investigates root cause, writes a fix, pushes it
- Handles reviewer comments: makes requested changes, pushes, and replies to the thread
- Skips duplicate or no-action events silently
- Asks the user (via `AskUserQuestion`) when a fix is ambiguous or architecturally significant

**Usage:**
```
> Watch PR #42 and fix any CI failures or review comments

# Claude calls subscribe_pr_activity internally, then waits.
# When a CI failure or review comment arrives, Claude automatically:
#   1. Investigates the failure/comment
#   2. Implements a fix
#   3. Pushes the commit
#   4. Replies to the reviewer if appropriate
```

You can also use community skills like [babysit-pr](https://gist.github.com/tilomitra/e0dca29b3a63b5b5aba62c1baeaa27b4) for more opinionated workflows, or combine with `/loop` for periodic status checks. This is one of the most powerful "set it and forget it" agentic workflows — particularly useful when waiting on slow CI pipelines or asynchronous code reviews.
