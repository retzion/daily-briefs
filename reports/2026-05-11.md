## Featured: /review Skill

The `/review` skill launches a structured pull request review directly from Claude Code. Instead of manually reading diffs, you can have Claude analyze an entire PR for correctness, security issues, performance concerns, and style.

**Why it's useful:** Code review is one of the highest-leverage activities in software development, but it's also time-consuming. `/review` gives you an opinionated, thorough second pair of eyes before you merge — catching issues that linters and tests miss, like logic errors, missing edge cases, or subtle security flaws.

**Usage:**
```
# Review the current branch's PR
/review

# Review a specific GitHub PR by number
/review 42
```

Claude examines every changed file, considers the PR as a whole (not just individual diffs), and returns a structured report with actionable findings. It pairs well with `/security-review` for defense-in-depth on sensitive changes.
