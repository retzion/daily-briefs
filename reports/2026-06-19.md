## Featured: Auto Mode Safety Guardrails

Today's v2.1.183 release adds automatic safety rails when Claude Code runs in auto mode. Three categories of dangerous operations are now blocked unless you explicitly asked for them:

1. **Destructive git commands** — `git reset --hard`, `git checkout -- .`, `git clean -fd`, `git stash drop` blocked when you didn't ask to discard local work
2. **Amend protection** — `git commit --amend` blocked when the commit wasn't made by the agent in the current session
3. **Infrastructure teardown** — `terraform destroy`, `pulumi destroy`, `cdk destroy` blocked unless you asked for the specific stack

**Why it matters:** Auto mode lets Claude Code execute tools without per-step approval, which is fast but risky. These guardrails prevent the most common categories of unrecoverable mistakes — accidentally nuking local changes, amending someone else's commit, or tearing down infrastructure.

**Usage:** These guardrails are enabled automatically in auto mode (no configuration needed). If you actually want to perform one of these operations, just say so explicitly — e.g., "reset the branch to match origin/main" or "destroy the staging stack."
