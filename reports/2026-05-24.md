## Featured: Agent Teams

**Agent Teams** is an experimental Claude Code feature that orchestrates multiple Claude Code sessions working together on a shared project — like having a small engineering squad where each member owns a piece of the work.

**Why it matters:** Subagents run in a strict parent-child hierarchy where each can only report back to its spawner. Agent Teams introduce **peer-to-peer communication** through a mailbox system and shared task list, so a frontend agent can directly tell a backend agent about an API contract change without routing through a coordinator.

**How it works:**
- One session acts as the **team lead** — it coordinates, assigns tasks, and synthesizes results
- Teammates work independently in their own context windows and can message each other directly
- Best for: cross-layer features (frontend + backend + tests), debugging with competing hypotheses, research tasks that benefit from diverse parallel investigation

**Quick start:**
```bash
# Enable the experimental feature
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1

# Launch Claude Code and ask it to form a team
claude

> Set up a team: one teammate for the React frontend, one for the
> Express API, and one for integration tests. Build the user
> settings page together.
```

Requires Pro, Max, Team, or Enterprise plan. ([docs](https://code.claude.com/docs/en/agent-teams))
