## Featured: Dynamic Workflows

Dynamic Workflows are Claude Code's new research-preview orchestration layer, shipped alongside Opus 4.8 on May 28. Instead of a single agent working sequentially, Claude writes a JavaScript orchestration script that fans work out across up to 1,000 subagents (max 16 running concurrently).

**Why it matters:** Tasks that would take hours in a single session — large-scale refactors, codebase-wide migrations, multi-file audits — can now be parallelized with adversarial verification built in. Jarred Sumner used dynamic workflows to port Bun from Zig to Rust: ~750k lines, 99.8% test pass rate, 11 days.

**How to use it:**
```
# Option 1: Set effort level for the whole session
/effort ultracode

# Option 2: Include "workflow" in any prompt
"Workflow: audit all API endpoints for auth gaps and generate a report"
```

Claude plans the work, breaks it into subtasks, spins up parallel agents, deploys adversarial agents to refute findings, and iterates until results converge — all while your session stays responsive.

**Caveats:** Consumes significantly more usage than normal sessions. Available on Max, Team, and Enterprise plans. Requires Claude Code v2.1.154+.
