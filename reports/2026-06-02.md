## Featured: `claude -p` (Pipe / Headless Mode)

Run Claude Code non-interactively in scripts, CI/CD pipelines, and automation. The `-p` flag (short for `--print`) processes your prompt, outputs the result to stdout, and exits — no TUI, no confirmations.

**Why it matters:** With Agent SDK billing splitting off on June 15, understanding headless usage is essential for budgeting. It's also the foundation for Claude Code in GitHub Actions, pre-commit hooks, and custom toolchains.

**Quick examples:**

```bash
# One-shot task
claude -p "Find and fix the bug in auth.py" --allowedTools "Read,Edit,Bash"

# Pipe data in
cat build-error.txt | claude -p "explain the root cause of this build error" > diagnosis.txt

# CI-friendly: skip local config, get structured output
claude -p "review this diff for security issues" --bare --output-format json
```

**Key flags:** `--bare` (ignore local config), `--output-format json|stream-json` (machine-readable output), `--allowedTools` (restrict available tools), `--max-turns` (limit agent loops).

[Docs](https://code.claude.com/docs/en/headless)
