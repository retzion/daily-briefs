## Featured: `alwaysLoad` MCP Config Option

New in Claude Code v2.1.122, the `alwaysLoad` option lets you pin specific MCP servers so their tools skip deferred tool-search and load into context immediately at session start.

**Why it matters:** Claude Code's tool-search system defers MCP tool schemas by default — great for context savings (~85-95%), but it means Claude has to "discover" tools before using them. For servers you reach for on every task (e.g., your database, CI, or project-specific tooling), `alwaysLoad: true` eliminates that extra step.

**Usage:**

```json
// .mcp.json
{
  "mcpServers": {
    "my-db": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@my-org/db-mcp"],
      "alwaysLoad": true
    }
  }
}
```

Individual tools can also opt in via `"anthropic/alwaysLoad": true` in the tool's `_meta` object. Use sparingly — each always-loaded tool consumes context that would otherwise be available for conversation.

Docs: [MCP configuration](https://code.claude.com/docs/en/mcp)
