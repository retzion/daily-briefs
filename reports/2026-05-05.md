## Featured: /hooks (Interactive Hook Editor)

Hooks are user-defined shell commands that execute at specific points in Claude Code's lifecycle — giving you deterministic control over behavior that doesn't rely on the LLM choosing to run them.

**Why it's useful:** Enforce project rules, auto-format on save, block dangerous operations, send notifications, or integrate with external tools — all without manual approval prompts.

**Quick usage:**

Run `/hooks` in Claude Code to open the interactive editor. Common patterns:

```jsonc
// Auto-format after every file write (in .claude/settings.json)
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": { "tool_name": "Write" },
        "command": "npx prettier --write \"$CLAUDE_FILE_PATH\""
      }
    ]
  }
}
```

**Key events:** `PreToolUse` (validate/block before action), `PostToolUse` (cleanup after action), `SessionStart`, `SessionEnd`, `UserPromptSubmit`, and more — 12 lifecycle events total.

**Pro tip:** Use `PreToolUse` to block writes to `.env` or production configs, making dangerous operations impossible rather than just discouraged.

Docs: [code.claude.com/docs/en/hooks](https://code.claude.com/docs/en/hooks-guide)
