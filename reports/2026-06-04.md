## Featured: `claude agents --json` (Programmatic Agent Monitoring)

The `claude agents` command has an interactive TUI for managing background sessions, but the `--json` flag unlocks programmatic access — useful for CI/CD pipelines, custom dashboards, and automation scripts.

**What it does:** Outputs structured JSON for every running, waiting, or completed background session, including session ID, status, name, elapsed time, and (new in v2.1.162) a `waitingFor` field that tells you exactly what a blocked session needs.

**Why it's useful:** If you're running multiple background agents (e.g., parallel PR reviews, test suites, or migrations), `--json` lets you build scripts that monitor progress, detect stuck sessions, and take action automatically.

**Quick example — poll until all agents finish:**
```bash
#!/bin/bash
while true; do
  waiting=$(claude agents --json | jq '[.[] | select(.status == "waiting")] | length')
  running=$(claude agents --json | jq '[.[] | select(.status == "running")] | length')
  echo "Running: $running, Waiting: $waiting"
  if [ "$running" -eq 0 ] && [ "$waiting" -eq 0 ]; then
    echo "All agents complete."
    break
  fi
  sleep 10
done
```

**Check what a stuck agent needs:**
```bash
claude agents --json | jq '.[] | select(.status == "waiting") | {name, waitingFor}'
```

Pairs well with `claude --bg` for fire-and-forget background tasks and `claude -p` for headless scripting.
