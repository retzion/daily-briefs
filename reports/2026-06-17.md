## Featured: MCP Tunnels (Research Preview)

MCP tunnels let Claude Managed Agents connect to Model Context Protocol servers inside your private network — databases, ticketing systems, internal APIs — without exposing them to the public internet.

**How it works:** You deploy a lightweight gateway inside your network that makes a single outbound encrypted connection to Anthropic infrastructure. No inbound firewall rules, no public endpoints. The agent sends tool calls through the tunnel; your MCP server responds as if it were a local client.

**Why it matters:** Enterprise teams can now give agents access to internal tools (Jira, Postgres, internal wikis) without risky network exposure. This is the missing piece for production agent deployments in regulated industries.

**Quick setup:**
1. Request access via the Claude Platform console (research preview gate)
2. Deploy the tunnel gateway in your network (`claude-tunnel` container image)
3. Register the tunnel endpoint with your Managed Agent configuration
4. The agent can now call tools on your private MCP server

Documentation: [MCP tunnels overview](https://platform.claude.com/docs/en/agents-and-tools/mcp-tunnels/overview) | [Quickstart](https://platform.claude.com/docs/en/agents-and-tools/mcp-tunnels/quickstart)
