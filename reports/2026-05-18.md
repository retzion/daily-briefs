## Featured: Session Start Hooks
**Session Start Hooks** let you run shell commands automatically whenever a Claude Code session begins — ensuring your environment is ready before you (or an agent) start working. This is especially useful for **cloud/web sessions** where containers are ephemeral and your project may need setup steps each time.

**Why it matters**: Without a startup hook, you manually run `npm install`, start dev servers, or configure environment variables at the beginning of every session. With a session start hook, all of that happens automatically — particularly valuable for CI-triggered routines and scheduled cloud agents that run unattended.

**Example** (in `.claude/settings.json`):
```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "npm install && npm run build"
          }
        ]
      }
    ]
  }
}
```

This runs `npm install && npm run build` every time a Claude Code session starts in the project. You can add multiple hooks for different setup steps — database migrations, env file generation, dependency checks, etc.

**Pro tip**: Combine with `CLAUDE.md` so Claude knows both the project context (via CLAUDE.md) and that the environment is pre-configured (via the hook).
