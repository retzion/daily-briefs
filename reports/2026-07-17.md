## Featured: `/health` (Installation Health Audit)

The `/health` command (alias: `/checkup`) goes beyond `/doctor`'s basic setup diagnostics. It performs a comprehensive audit of your Claude Code installation's efficiency and hygiene:

- **Finds unused skills, MCP servers, and plugins** and reports their context cost — every loaded skill eats tokens even if you never invoke it
- **Deduplicates CLAUDE.md files** across your project hierarchy (project root, subdirectories, home dir)
- **Proposes trimming CLAUDE.md content** that's redundant or overly verbose
- **Flags slow hooks** that add latency to every tool call
- Reports findings first, asks for confirmation before changing anything

**Usage:**
```
/health
```

**Why it matters:** As projects accumulate skills, plugins, and MCP servers, context window bloat silently degrades performance and increases costs. `/health` is a one-command way to reclaim that headroom. Run it periodically or whenever Claude starts feeling sluggish.
