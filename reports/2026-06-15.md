## Featured: `/run` Skill (Launch & Drive Your App)

The `/run` skill launches your project's application and drives it to visually confirm a change works — going beyond type-checking and tests to verify actual feature behavior.

**What it does:** When invoked, `/run` first checks for a project-specific skill that covers launching the app. If none exists, it falls back to built-in patterns for the project type (CLI, server, TUI, Electron, browser app, library). It starts the app, navigates to the relevant feature, and observes behavior.

**Why it's useful:** Tests verify code correctness, not feature correctness. `/run` closes the gap — especially useful after UI changes, API route additions, or config tweaks where you want to see the real thing running before pushing.

**Quick usage:**
```
> /run
# Claude Code detects project type, starts the dev server,
# opens the app, and reports what it observes.

> /run check the new settings page renders correctly
# Targeted: launches the app and navigates to a specific feature.
```

**Compared to `/verify`:** `/verify` is for confirming a specific change works against expected behavior. `/run` is broader — it's "start the app and show me it's working," useful for smoke-testing or exploring a feature you just built.
