## Featured: `/plugin list`
New in v2.1.163 (June 4). Lists all installed Claude Code plugins with optional filtering.

**Why it's useful**: As the plugin ecosystem grows, it's easy to lose track of what's installed, what's active, and what's just sitting there. This command gives you a quick inventory without digging through config files.

**Usage**:
```bash
# List all installed plugins
/plugin list

# Show only enabled plugins
/plugin list --enabled

# Show only disabled plugins
/plugin list --disabled
```

Pair with `/plugins` (the interactive plugin manager) for enable/disable/remove actions. Use `/plugin list --disabled` to audit plugins you installed but forgot about — they still consume startup time if their MCP servers are configured with `alwaysLoad`.
