## Featured: Nested Sub-Agents (5 Levels Deep)

As of v2.1.172 (June 10), Claude Code sub-agents can now spawn their own sub-agents, up to 5 levels deep. This lifts the previous hard restriction where "subagents cannot spawn other subagents."

**Why it matters:** Complex tasks can now be decomposed hierarchically. A root agent can delegate to specialists, which can further delegate leaf tasks — enabling patterns like triage → analysis → targeted fix without flattening everything into a single context window.

**Token cost warning:** Nesting multiplies tokens geometrically, not linearly. A common mitigation is model tiering: Opus at root, Sonnet at level 1, Haiku from level 2+. The ~10x pricing gap between Opus and Haiku input tokens compounds across every nested call.

**Quick example in a custom agent definition (`.claude/agents/deep-review.md`):**
```markdown
You are a code review coordinator. For each changed file:
1. Spawn a sub-agent to analyze the diff
2. That sub-agent may spawn its own sub-agent for complex functions
3. Collect all findings and produce a unified review

Use model tiering: sonnet for file-level, haiku for function-level.
```

**Limits:** Max 5 levels deep. Each level carries its own system prompt and context. The parent only sees the leaf's summary — intermediate context is discarded.
