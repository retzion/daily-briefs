## Featured: The `Monitor` Tool
A newer Claude Code built-in that streams events from a long-running background process — each stdout line arrives as a notification, so the agent can react to log output, test failures, or deploy progress *without* polling.

**Why it's useful:** Replaces the old pattern of `sleep` + re-reading a log file. You start a process in the background once, then subscribe with `Monitor` using an `until`-style condition; Claude unblocks the moment the condition is met. Plugin authors can now also ship background monitors declaratively via the `monitors` manifest key (today's changelog).

**Quick example:**
```
# 1. Kick off a long-running process in background
Bash(command="npm run dev", run_in_background=true)

# 2. Stream its events until the dev server says it's ready
Monitor(pattern="ready in \\d+ms", timeout=60000)

# 3. Continue with browser tests once Monitor returns
```
For one-shot "wait until done," keep using `Bash(run_in_background=true)` — you'll be notified on completion. Use `Monitor` when you need *streaming* visibility or a targeted condition mid-stream.
