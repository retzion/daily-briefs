## Featured: Dynamic Workflow Size Setting

New in v2.1.202, the `workflow.dynamicSize` setting (accessible via `/config`) lets you control how many agents Claude Code spawns when building dynamic workflows with `/effort ultracode` or the Workflow tool.

**Why it's useful:** Workflows are powerful but token-hungry. A large code review workflow might spawn 20+ agents for parallel dimension-scanning and adversarial verification. Sometimes that's exactly what you want; other times you'd rather trade thoroughness for speed and cost. This setting gives you the dial.

**Options:**
- **small** — Fewer agents, faster and cheaper. Good for quick checks or when you're iterating rapidly.
- **medium** (default) — Balanced coverage. Suitable for most tasks.
- **large** — Maximum agent fleet. Use for thorough audits, security reviews, or when correctness matters more than cost.

**Quick usage:**
```bash
# Set via /config in a session
> /config

# Or set inline
> /config workflow.dynamicSize=large

# Then run a workflow-heavy task
> /effort ultracode Review this PR for security issues
```

The setting persists across sessions and can be overridden per-project in `.claude/settings.json`.
