## Featured: Session Resource Caps

New in v2.1.212 (July 17), Claude Code supports session-wide limits on web searches and subagent spawns via two environment variables. These prevent runaway loops where an agent endlessly searches or spawns sub-agents, burning through your usage budget.

**Environment variables:**
- `CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION` — caps total WebSearch calls per session (default: 200)
- `CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION` — caps total subagent spawns per session (default: 200)

**Why it matters:** Complex workflows involving `/deep-research`, `/loop`, or multi-agent orchestration can occasionally spiral into hundreds of searches or agent spawns. These caps provide a safety net without limiting normal usage.

**Usage example:**
```bash
# Tighter caps for cost-conscious runs
export CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION=50
export CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION=30
claude

# Or set per-invocation
CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION=10 claude -p "research this topic"
```

When a cap is hit, Claude Code informs you and stops spawning new searches/agents rather than silently failing. Adjust limits up or down depending on your workflow needs.
