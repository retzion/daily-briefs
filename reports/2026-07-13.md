## Featured: Custom Agent Types (`.claude/agents/`)

Define reusable subagent profiles as Markdown files with YAML frontmatter in `.claude/agents/` (project-scoped) or `~/.claude/agents/` (user-scoped). Each file specifies the agent's model, available tools, and system prompt — giving you fine-grained control over cost, capabilities, and behavior.

**Why it's useful:** Instead of relying on generic subagents for every task, you can create purpose-built agents — a security auditor on Opus with read-only tools, a doc maintainer on Haiku with just file ops, a test runner on Sonnet with Bash access. Team members get the same agents when they check `.claude/agents/` into version control.

**Example** — create `.claude/agents/security-auditor.md`:

```markdown
---
name: security-auditor
description: Read-only security review agent
model: opus
tools:
  - Read
  - Grep
  - Glob
---
You are a security auditor. Analyze code for OWASP Top 10
vulnerabilities, hardcoded secrets, and unsafe deserialization.
Report findings with file, line, severity, and remediation.
```

Then reference it in workflows or the Agent tool with `agentType: "security-auditor"`. The agent runs in its own context window with only the tools you specified.

**Tip:** Three to five concurrent subagents is the sweet spot. Route exploratory tasks to Sonnet/Haiku to cut cost without losing quality on well-scoped work. ([docs](https://code.claude.com/docs/en/sub-agents))
