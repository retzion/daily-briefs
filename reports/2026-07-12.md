## Featured: In-App Browser (Claude Code Desktop)

Claude Code Desktop now includes a built-in browser that lets Claude open, read, click through, and interact with any web page — right inside the app. Unlike the Chrome extension (which uses your existing browser sessions and logins), the in-app browser runs in a sandboxed, isolated profile with no saved history or credentials.

**Why it matters:** Before this, verifying a UI change against external docs or a design spec required switching contexts. Now Claude can pull up a Figma mock, a documentation page, or a staging URL and reference it directly while coding — closing the feedback loop without leaving the session.

**Quick usage:**
- `Ctrl+Shift+B` (Windows/Linux) or `Cmd+Shift+B` (macOS) to open the browser pane
- Claude can also open links shared in conversation directly in the pane
- First interaction with any site prompts: Allow once / Always allow / Deny
- Sessions are configurable — choose whether browsing state persists between sessions

**When to use the Chrome extension instead:** When you need Claude to interact with pages where you're logged in (e.g., your actual GitHub, Jira, or internal tools).
