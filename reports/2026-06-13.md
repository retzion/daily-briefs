## Featured: `enforceAvailableModels` Managed Setting

New in v2.1.175 (June 12), this enterprise admin setting locks down which models Claude Code users can access — and now constrains the Default model slot too.

**Why it matters today:** With Fable 5 suddenly disabled by government directive, teams that had it as their default hit immediate breakage. `enforceAvailableModels` with a curated allowlist ensures graceful fallback.

**Configuration (managed settings):**
```json
{
  "availableModels": ["claude-opus-4-8", "claude-sonnet-4-6", "claude-haiku-4-5"],
  "enforceAvailableModels": true
}
```

When enabled:
- Default resolving to a disallowed model falls back to the first model in the list
- Users cannot expand the allowlist via user or project settings
- `/model` picker only shows allowed models

**Use case:** Control cost (block expensive models), enforce compliance (approved models only), or handle availability disruptions like the Fable 5 suspension.
