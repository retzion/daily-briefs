## Featured: `fallbackModel` Setting
New in v2.1.166 (June 6). Configures up to three fallback models that Claude Code tries in order when your primary model is overloaded or returns an unexpected error.

**Why it's useful**: Model outages and rate limits are the most common source of interrupted agentic sessions. Instead of your session halting with an error, it silently falls back to your next-best model and keeps working. Especially valuable for long-running background agents and `claude -p` pipelines that can't afford to stop.

**Usage**:
```jsonc
// In .claude/settings.json or settings.local.json
{
  "fallbackModel": [
    "claude-sonnet-4-6",      // first fallback
    "claude-haiku-4-5-20251001" // second fallback
  ]
}
```

Or via CLI flag:
```bash
claude --fallback-model claude-sonnet-4-6
```

Auth errors, rate limits, and transport errors still surface immediately — the fallback only triggers on unexpected non-retryable API errors. Claude Code retries once on the fallback model, then promotes the error if it persists.
