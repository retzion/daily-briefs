## Featured: /fewer-permission-prompts Skill

Permission prompts are the most common friction point in Claude Code sessions — every unfamiliar Bash command or MCP tool call triggers a confirmation dialog. The `/fewer-permission-prompts` skill automates the fix.

**What it does:** Scans your recent session transcripts for commonly executed read-only Bash commands and MCP tool calls, then proposes a prioritized allowlist to add to your project's `.claude/settings.json`. It only suggests commands that are safe and read-only — no destructive operations.

**Why it's useful:** Instead of manually curating permission rules, you get a data-driven allowlist based on your actual workflow. New team members benefit immediately from the shared project settings.

**Usage:**
```
/fewer-permission-prompts
```

It analyzes your transcripts and outputs a proposed settings block like:

```json
{
  "permissions": {
    "allow": [
      "Bash(npm test:*)",
      "Bash(git status)",
      "Bash(git diff:*)",
      "Bash(grep:*)",
      "mcp__github__pull_request_read"
    ]
  }
}
```

Review the suggestions, approve, and your next session will flow with far fewer interruptions.
