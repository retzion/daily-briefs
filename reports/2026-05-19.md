## Featured: /resume for Background Sessions
The `/resume` slash command now includes **background sessions** in its session picker — a quality-of-life improvement shipping in today's v2.1.144.

**Why it matters**: Previously, sessions started with `claude --bg` or dispatched from agent view lived in a separate world — you could only check them via `claude agents`. Now `/resume` shows them alongside interactive sessions, tagged with a `bg` label, so you have a single unified list of all your work.

**Bonus**: Background subagent completion notifications now include **elapsed duration** (e.g. "Agent completed · 3h 2m 5s"), making it easy to gauge how long autonomous work actually took.

**Usage**:
```bash
# Start a background session
claude --bg "refactor the auth module"

# Later, resume it interactively
claude
> /resume
# Background sessions now appear in the list with a [bg] marker
```

This pairs well with `claude agents` (the dashboard view) — use `claude agents` for a bird's-eye overview, `/resume` when you want to jump into a specific session.
