## Featured: `/skill-creator` (Custom Skill Authoring)

Claude Code's `/skill-creator` skill lets you build, test, and optimize your own skills — extending Claude Code's capabilities with project-specific or workflow-specific automation.

**What it does:** Guides you through creating a new skill from scratch, editing existing skills, running evals to test triggering accuracy, and benchmarking performance with variance analysis. Skills are stored as markdown files in `.claude/skills/` and can include trigger descriptions, instructions, and examples.

**Why it matters:** The built-in skills cover common workflows, but every team has unique patterns — deploy scripts, review checklists, data pipelines, compliance checks. Custom skills let you encode those patterns once and invoke them by name, turning tribal knowledge into repeatable automation.

**Quick usage:**
```
> /skill-creator

# Then follow the interactive flow:
# 1. "Create a new skill" → name it, describe when it triggers, write instructions
# 2. "Run evals" → test whether various prompts correctly trigger your skill
# 3. "Optimize" → refine the description for better triggering accuracy

# Result: a .claude/skills/your-skill.md file that any team member can invoke
```

Custom skills compose with other Claude Code features — they can reference MCP tools, call other skills, and use hooks for pre/post automation.
