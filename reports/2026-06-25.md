## Featured: `claude mcp login` / `claude mcp logout`

New CLI commands (v2.1.186+) for authenticating MCP servers without opening the interactive `/mcp` panel inside a session. Especially useful for headless setups, SSH sessions, CI pipelines, and remote containers.

**Usage:**
```bash
# Authenticate an MCP server via OAuth flow
claude mcp login my-server

# Clear stored credentials
claude mcp logout my-server
```

**Headless support (v2.1.191+):** When no browser is available (SSH, remote container), the command prints the authorization URL instead of trying to open a browser. Open the URL on your local machine, complete the OAuth flow, then paste the redirect URL back at the prompt. Use `--no-browser` to force URL-paste mode even when a browser is detected.

**Why it matters:** Before these commands, authenticating MCP servers required an interactive Claude Code session with the `/mcp` menu. Now you can pre-authenticate servers as part of setup scripts, dotfile bootstrapping, or session-start hooks — making MCP-dependent workflows reliable in automated and remote environments.
