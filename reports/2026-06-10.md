## Featured: `/cd` Command (In-Session Directory Switching)

Added in v2.1.169, the `/cd` command lets you change Claude Code's working directory mid-session **without breaking the prompt cache**. Previously, switching projects meant starting a fresh session and losing all accumulated context.

**Why it's useful:** Many workflows span multiple directories — a monorepo with separate frontend/backend folders, a main project plus a shared library, or switching between repos during a debugging session. `/cd` keeps your conversation history and cached context intact while pivoting to a new location.

**Usage:**
```
/cd ../backend
/cd ~/projects/shared-lib
/cd /absolute/path/to/other/repo
```

After switching, Claude Code operates in the new directory for all file operations, glob searches, and shell commands. Your prompt cache is preserved, so you don't pay the cold-start cost of re-establishing context.
