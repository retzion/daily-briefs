## Featured: Pinned Background Sessions (Ctrl+T)

When running multiple background agents via `claude agents`, some sessions are more important than others. **Pinned sessions** solve this by marking a background session as high-priority so the system treats it differently:

- **Stays alive when idle** — non-pinned sessions may be reaped during inactivity, but pinned ones persist
- **Auto-restarts for updates** — when a new Claude Code version ships, pinned sessions restart in place rather than being discarded
- **Last to shed** — under memory pressure, non-pinned sessions are shed first

**Usage**: Open the agents view with `claude agents`, navigate to a session, and press `Ctrl+T` to toggle its pinned status. Useful for long-running watchers (CI monitors, PR babysitters) that you don't want interrupted.
