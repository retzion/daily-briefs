## Featured: Fullscreen Rendering (`/tui fullscreen`)

Claude Code's fullscreen rendering mode draws the interface on the terminal's **alternate screen buffer** (like vim or htop), only rendering messages currently visible. The result:

- **No flicker** — eliminates the jumping/flashing common in long sessions
- **Flat memory usage** — performance stays constant as conversations grow
- **Mouse support** — click to position cursor, click suggestions to accept, click tool results to expand/collapse, click URLs to open, wheel to scroll

**Enable it:**
```bash
/tui fullscreen          # toggle in current session
# or set permanently:
export CLAUDE_CODE_NO_FLICKER=1
```

**New in v2.1.195:** If you want wheel scrolling but mouse clicks interfere with your workflow (e.g., tmux copy mode, terminal selection), set:
```bash
export CLAUDE_CODE_DISABLE_MOUSE_CLICKS=1
```
This keeps wheel scroll active while disabling click/drag/hover — the best of both worlds.

**Tip for tmux users:** Add `set -g mouse on` to `~/.tmux.conf` for wheel scrolling support. Note: fullscreen is incompatible with iTerm2's `tmux -CC` integration mode.
