## Claude Code
- **v2.1.166–168 shipped June 6** — headline features include `fallbackModel` setting (up to three fallback models when primary is overloaded), glob pattern support in deny rules (`"*"` denies all tools), and `MAX_THINKING_TOKENS=0` / `--thinking disabled` to turn off thinking on models that think by default ([changelog](https://code.claude.com/docs/en/changelog))
- **Cross-session message hardening** — messages relayed via `SendMessage` from other Claude sessions no longer carry user authority; receivers refuse relayed permission requests and auto mode blocks them
- **`claude update` now announces target version** before downloading instead of going silent
- **`claude agents` URL filter** — typing a URL into the agent list now filters to the session whose first prompt contained it
- **Bug fixes**: recurring "image could not be processed" error resolved; remote sessions no longer get permanently stuck on brief backend disruptions; JetBrains terminal flickering fixed; Shift+non-ASCII characters (Kitty protocol) fixed; orphaned `--bg-pty-host` processes at 100% CPU on macOS fixed; `/doctor` no longer shows contradictory "Not inside a remote session" check when run remotely ([GitHub releases](https://github.com/anthropics/claude-code/releases))

## Featured: `/doctor` (Setup Diagnostics)

`/doctor` is Claude Code's built-in health check that audits your entire setup in seconds. Run it inside a session to diagnose issues before they derail your work.

**What it checks:**
- Installation type (npm global, local, or native) and upgrade path
- API connectivity and authentication state
- MCP server health and JSON config validity
- Hook syntax and shell integration
- Context usage and session state

Each check is color-coded: green (pass), yellow (warning), red (error). After the report, press `f` to let Claude auto-fix the issues it found.

**Usage:**
```
# Inside a Claude Code session:
/doctor

# If Claude Code won't start at all:
claude doctor
```

**When it's useful:** After upgrading Claude Code, adding a new MCP server, changing auth providers, or when something just feels off. It's faster than grepping logs and more reliable than guessing.
