## Featured: Artifact Runtime Capabilities

When you publish an HTML page as a Claude Code Artifact, you can declare **runtime capabilities** that go beyond static content — live data, shared state between viewers, and self-updating behavior.

**What it is:** The `capabilities` parameter on the Artifact tool declares what runtime features a published page needs. Before using any capability, load the `artifact-capabilities` skill to see what's available and how to use the `window.claude.*` APIs.

**Why it's useful:** Instead of publishing a static snapshot that goes stale, your artifacts can connect to MCP connectors for real-time data. A team dashboard can show current metrics, a monitoring page can refresh itself, and multiple viewers can see shared state — all without external CDNs or API calls.

**Quick usage:**
```
# 1. Load the skill first to see available capabilities
/artifact-capabilities

# 2. Publish with capabilities declared
Artifact({
  file_path: "dashboard.html",
  favicon: "📊",
  description: "Live project metrics dashboard",
  capabilities: { "live-data": { connector: "github" } }
})
```

The page's JavaScript calls `window.claude.*` methods to fetch live data — everything routes through the artifact runtime. Omitting `capabilities` on a redeploy keeps existing declarations; passing `{}` clears them.
