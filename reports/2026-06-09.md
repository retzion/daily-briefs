## Featured: `--safe-mode` (Troubleshooting Flag)

When Claude Code misbehaves and you're not sure if it's a core issue or something in your config, `--safe-mode` is the fastest way to find out. It launches Claude Code with **all customizations disabled**: CLAUDE.md files, plugins, skills, hooks, and MCP servers are all skipped.

**Why it's useful:** As Claude Code configs grow more complex — custom hooks, multiple MCP servers, third-party plugins — debugging becomes harder. Safe mode gives you a clean baseline in one flag, similar to browser safe mode or macOS Safe Boot.

**Usage:**
```bash
# Via CLI flag
claude --safe-mode

# Via environment variable
CLAUDE_CODE_SAFE_MODE=1 claude
```

If the problem disappears in safe mode, you know it's a config issue — re-enable customizations one at a time to isolate the culprit.
