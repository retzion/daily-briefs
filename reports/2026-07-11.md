## Featured: `/commit-push-pr` (One-Command Ship Workflow)

The `/commit-push-pr` slash command bundles committing, pushing, and PR creation into a single step. Run it after finishing a feature and Claude will:

1. Create a feature branch (if you're on main)
2. Stage and commit with a generated message
3. Push to your remote
4. Open a PR with summary and test plan
5. Return the PR URL

**What's new (v2.1.206):** It now auto-allows `git push` to your repo's configured push remote (`remote.pushDefault`, or the sole remote), so you skip the permission prompt for non-origin remotes.

**Usage:**
```
> /commit-push-pr
```

It uses inline bash to pre-compute `git status` and diff stats, keeping the interaction fast. The command lives at `.claude/commands/commit-push-pr.md` — you can customize it or create your own workflow commands in that directory.

**Tip:** Use `/commit` for intermediate saves during development, then `/commit-push-pr` when you're ready to ship. Pairs well with `/code-review` to self-review before pushing.
